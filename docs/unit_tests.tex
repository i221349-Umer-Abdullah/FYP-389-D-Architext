\section{Unit Testing}
% ----------------------------------------------------------

Unit testing was conducted on the six core backend modules of ArchiText: the NLP Adapter, Spec Converter, Plot Constraint Fitter, Request/Response Schemas, Job Manager, and Plot Unit Converter. A total of 56 test cases were designed and executed programmatically against the live codebase. Each test was constructed with a clearly defined precondition, a specific set of inputs, and an unambiguous expected outcome so that results could be verified objectively. Fifty-five of the 56 test cases passed on the first run. The single failing test, TC-PF-08, exposed a genuine algorithmic limitation in the Plot Constraint Fitter that was subsequently diagnosed and corrected; all 56 tests pass in the final codebase.

\medskip
\noindent\textbf{Note on TC-PF-08 (Plot Fitter — Overlap Resolution):}
Test case TC-PF-08 was designed to verify that the push-apart algorithm resolves overlapping rooms after plot fitting. The test submitted three identically-sized rooms all positioned at the same coordinate \((0, 0)\) on a 15\,m\,$\times$\,12\,m plot. The test revealed that the uniform scaling step in Step 2 of the fitting algorithm computed the layout's bounding box as a single room's footprint (since all three rooms were coincident), and scaled that single footprint to fill the available area. Each room consequently expanded to 11.3\,m\,$\times$\,11.3\,m — the full height of the available plot — leaving the push-apart algorithm with no space to physically separate them. Two fixes were implemented in \texttt{backend/core/plot\_fitter.py}: (1) \textbf{Step 3.5 — Grid-based scale-down}: after enforcing minimum sizes, if any room exceeds its fair share of a $\lceil\sqrt{N}\rceil \times \lceil\sqrt{N}\rceil$ grid, it is scaled down so all $N$ rooms can physically pack into the plot; (2) \textbf{Step 3.6 — Position degeneracy breaker}: rooms sharing the exact same $(x, y)$ coordinate are pre-placed in a grid arrangement before the push-apart runs, giving the algorithm distinct starting positions to work from. After applying these fixes, TC-PF-08 passed, and all previously passing tests continued to pass. This fix also improves the system's robustness against mode-collapse output from the CVAE-GNN model, where generated rooms can converge to near-identical positions.
